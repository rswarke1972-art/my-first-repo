Trigger

A Trigger is a stored PL/SQL block that automatically executes in response to certain events on a table, such as INSERT, UPDATE, or DELETE.

Types of Triggers

1. BEFORE Trigger

  • Executes before the event occurs

2. AFTER Trigger

  • Executes after the event occurs

3. INSTEAD OF Trigger

  • Used mainly with views

Example

CREATE OR REPLACE TRIGGER student_trigger BEFORE INSERT ON Student FOR EACH ROW BEGIN DBMS_OUTPUT.PUT_LINE('New record inserted'); END;

Advantages

  • Automatic execution
  • Maintains data integrity
  • Improves security

Conclusion

Triggers automatically execute when specific database events occur, ensuring control and integrity.
