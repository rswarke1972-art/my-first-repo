TCL (Transaction Control Language)

TCL (Transaction Control Language) is used to manage transactions in a database. It ensures data consistency and integrity.

Main TCL Commands

1. COMMIT

  • Saves all changes permanently

2. ROLLBACK

  • Undoes changes since last COMMIT

3. SAVEPOINT

  • Creates a point to which rollback can occur

4. SET TRANSACTION

  • Specifies transaction properties

Transaction Properties (ACID)

  • Atomicity
  • Consistency
  • Isolation
  • Durability

Conclusion

TCL commands control transactions and maintain database reliability.
