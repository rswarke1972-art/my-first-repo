Relational Integrity

Relational Integrity refers to rules that ensure accuracy and consistency of data in relational databases. These rules prevent invalid data from being stored.

Types of Relational Integrity

1. Entity Integrity

  • Primary key cannot be NULL
  • Ensures unique identification of records

2. Domain Integrity

  • Attribute values must be within defined domain

3. Referential Integrity

  • Foreign key must match primary key in referenced table

Importance

  • Prevents inconsistent data
  • Maintains database reliability

Conclusion

Relational Integrity ensures correctness and consistency of data in a relational database.
