Atomic Domains and First Normal Form (1NF)

Atomic Domain means that each attribute in a table contains indivisible (atomic) values. This concept leads to First Normal Form (1NF).

Atomic Domain

  • Each field contains single value
  • No repeating groups

First Normal Form (1NF)

  • Table must have atomic values
  • No multi-valued attributes
  • Each row must be unique

Example (Not in 1NF)

Student (Roll_No, Name, Phone_Numbers)

If Phone_Numbers contains multiple values → Not 1NF

Converted to 1NF

Student (Roll_No, Name, Phone_Number)

Conclusion

1NF ensures atomic values and removes repeating groups from a relation.
