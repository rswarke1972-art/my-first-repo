Second Normal Form (2NF)

A relation is in Second Normal Form (2NF) if it is already in First Normal Form (1NF) and has no partial dependency.

Partial Dependency

  • Occurs when a non-prime attribute depends on part of a composite key

Conditions for 2NF

  • Relation must be in 1NF
  • No partial dependency

Example

Student_Course (Roll_No, Course_ID, Course_Name)

  • Composite Key: (Roll_No, Course_ID)
  • If Course_Name depends only on Course_ID → Partial dependency

Solution

  • Split into separate tables

Conclusion

2NF removes partial dependencies and improves database structure.
