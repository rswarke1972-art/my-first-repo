Attributes and Domains

In the relational model, attributes represent columns of a table, and domains define the set of valid values for those attributes.

Attributes

  • Represent properties of an entity
  • Each column in a table is an attribute
  • Example: Roll_No, Name, Age

Types of Attributes

  • Simple Attribute
  • Composite Attribute
  • Single-valued Attribute
  • Multi-valued Attribute

Domain

  • Set of permissible values for an attribute
  • Example: Age → Integer between 0 and 120

Importance

  • Maintains data integrity
  • Prevents invalid data entry

Conclusion

Attributes define the structure of a relation, while domains restrict the valid values.
