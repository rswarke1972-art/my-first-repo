Codd's Rules

Codd's Rules are a set of 13 rules proposed by E.F. Codd to define what a true relational database management system (RDBMS) should be.

Important Codd's Rules

Rule 0

  • System must manage database using relational model

Rule 1 – Information Rule

  • All data stored in tables

Rule 2 – Guaranteed Access Rule

  • Each data item accessible using table name, primary key, and column name

Rule 3 – Systematic Treatment of NULL Values

  • NULL values supported for missing data

Rule 4 – Dynamic Online Catalog

  • Database description stored as tables

Rule 5 – Comprehensive Data Sublanguage

  • Support for DDL, DML, security, and integrity

Rule 6 – View Updating Rule

  • Views should be updatable

Conclusion

Codd's Rules define the standards for a true relational database system.
