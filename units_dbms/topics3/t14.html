Referential Integrities

Referential Integrity ensures that relationships between tables remain consistent. It maintains valid references between primary and foreign keys.

Definition

  • A foreign key must match a primary key value in the referenced table
  • Or it must be NULL (if allowed)

Example

Student (Roll_No, Name, Course_ID) Course (Course_ID, Course_Name)

  • Course_ID in Student is a foreign key
  • It must exist in Course table

Actions on Referential Integrity

  • CASCADE
  • SET NULL
  • RESTRICT

Importance

  • Prevents orphan records
  • Maintains data consistency

Conclusion

Referential Integrity ensures proper relationships between related tables.
