Entity Relationship Model

The Entity Relationship (ER) Model is a high-level conceptual data model used to describe the structure of a database. It represents data using entities, attributes, and relationships.

Main Components of ER Model

1. Entity

  • Represents a real-world object
  • Example: Student, Teacher, Course

2. Attribute

  • Describes properties of an entity
  • Example: Name, Roll_No, Age

3. Relationship

  • Shows connection between entities
  • Example: Student enrolls in Course

Types of Entities

  • Strong Entity
  • Weak Entity

Importance

  • Provides clear database structure
  • Easy to understand and design
  • Helps in converting into tables

Conclusion

The ER Model is widely used in database design to represent data and relationships visually.
