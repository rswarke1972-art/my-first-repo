Database Design and ER Model

Database Design is the process of creating a structured database that satisfies user requirements. The ER (Entity-Relationship) Model is used to represent data and relationships visually.

1. Entity

  • Represents a real-world object
  • Example: Student, Employee, Course

2. Attributes

  • Properties of an entity
  • Example: Student_ID, Name, Age

3. Relationships

  • Defines association between entities
  • Example: Student enrolls in Course

4. Constraints

  • Rules applied to data
  • Maintain data integrity

5. Keys

  • Primary Key – Uniquely identifies a record
  • Foreign Key – References primary key of another table

Importance

  • Improves database structure
  • Reduces redundancy
  • Ensures data consistency

Conclusion

Database design using the ER Model helps create organized and efficient database systems.
