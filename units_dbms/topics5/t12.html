CAP Theorem and BASE Properties

The CAP Theorem explains the limitations of distributed database systems. BASE properties are used in NoSQL systems as an alternative to ACID.

CAP Theorem

A distributed system can guarantee only two out of the following three properties:

  • Consistency (C) – All nodes see the same data
  • Availability (A) – System always responds to requests
  • Partition Tolerance (P) – System works despite network failure

BASE Properties

  • Basically Available – System guarantees availability
  • Soft State – Data may change over time
  • Eventually Consistent – Data becomes consistent eventually

Difference from ACID

  • ACID ensures strict consistency
  • BASE focuses on availability and scalability

Conclusion

CAP Theorem defines trade-offs in distributed systems, while BASE provides flexible consistency models for NoSQL databases.
