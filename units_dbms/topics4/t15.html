Serial Schedule

A Serial Schedule is a schedule in which transactions are executed one after another without overlapping.

Characteristics

  • No interleaving of operations
  • One transaction completes before another begins
  • Maintains database consistency

Example

T1: Read(A) → Write(A) → Commit T2: Read(B) → Write(B) → Commit

Here, T2 starts only after T1 finishes.

Advantages

  • Simple to implement
  • No concurrency problems

Disadvantages

  • Low performance
  • No parallel execution

Conclusion

Serial schedules ensure correctness but reduce system efficiency.
