Concurrency Control: Need, Locking Methods

Concurrency Control is used to manage simultaneous execution of transactions without causing data inconsistency. It ensures isolation and serializability.

Need for Concurrency Control

  • Prevent lost updates
  • Avoid dirty reads
  • Maintain data consistency
  • Ensure serializability

Locking Methods

1. Shared Lock (S)

  • Allows multiple transactions to read data
  • No transaction can write

2. Exclusive Lock (X)

  • Allows transaction to read and write
  • No other transaction can access data

3. Two-Phase Locking (2PL)

  • Growing phase: Acquire locks
  • Shrinking phase: Release locks

Conclusion

Concurrency control ensures safe and consistent execution of multiple transactions.
