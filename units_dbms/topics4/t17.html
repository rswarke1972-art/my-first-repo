Cascaded Aborts

Cascaded Abort occurs when one transaction fails and causes other dependent transactions to fail. This happens when transactions read uncommitted data.

Cause of Cascaded Abort

  • Transaction T1 writes data
  • Transaction T2 reads that uncommitted data
  • If T1 aborts, T2 must also abort

Example

T1: Write(A) T2: Read(A) If T1 aborts → T2 must abort

Problem

  • Multiple rollbacks
  • Reduced performance

Solution

  • Use strict schedules
  • Allow reading only committed data

Conclusion

Cascaded aborts occur due to reading uncommitted data and can be avoided using strict scheduling.
