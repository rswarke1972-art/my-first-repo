Write Policy

Write policy defines how data is written into cache and main memory when the CPU performs a write operation.

Main Write Policies

1. Write Through

In write-through, data is written to both cache and main memory at the same time.

  • Simple and safe
  • Slower due to frequent memory writes

2. Write Back (Write-Behind)

In write-back, data is written only to cache. Main memory is updated later when the cache block is replaced.

  • Faster because it reduces memory writes
  • Needs additional hardware for tracking modified blocks

Conclusion

Write-back is faster but complex, while write-through is simple and reliable. Modern systems use a combination of both depending on requirements.
