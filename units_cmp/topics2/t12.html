Locality of Reference

Locality of reference is a principle which states that programs tend to access a small portion of memory repeatedly for a short period of time. This property helps in improving cache performance.

Types of Locality of Reference

1. Temporal Locality

If a memory location is accessed, it is likely to be accessed again soon.

  • Example: Variables inside loops
  • Recently used data is reused

2. Spatial Locality

If a memory location is accessed, nearby memory locations are likely to be accessed soon.

  • Example: Accessing elements of an array sequentially
  • Adjacent memory locations are used

3. Sequential Locality

Instructions are generally executed sequentially, one after another.

Importance of Locality

  • Improves cache efficiency
  • Reduces memory access time
  • Increases overall system performance

Conclusion

Locality of reference is the main reason why cache memory works effectively. By storing recently and closely used data, cache reduces the need to access slower main memory.
