Memory Hierarchy

Memory hierarchy is a structure that uses different types of memory devices arranged in levels. Each level has different speed, size, and cost. The main idea is to provide fast access to frequently used data while keeping large capacity at a low cost.

Levels of Memory Hierarchy

  • Registers – Smallest and fastest memory inside CPU
  • Cache Memory – Very fast memory placed near CPU
  • Main Memory (RAM) – Larger but slower than cache
  • Secondary Storage – HDD, SSD (large capacity, slower)
  • Offline Storage – Tapes, external drives (very large capacity)

Why Memory Hierarchy is Used?

It is impossible to use only fast memory because it is very expensive. Memory hierarchy balances speed and cost by using small fast memory with large slow memory.

Key Concepts

  • Hit – Data is found in faster memory (cache)
  • Miss – Data is not found in cache and must be fetched from main memory
  • Locality – Program uses data located near previously used data

Conclusion

Memory hierarchy improves performance by keeping frequently used data in faster memory levels while storing the rest in slower, larger memory.
